# Non-Negotiable System Rules

These 15 rules are non-negotiable product constraints. Every feature, surface, and backend system must comply.

## The 15 Rules

| # | Rule |
|---|------|
| 1 | Every vehicle must show all applicable acquisition paths. |
| 2 | Every path must show full outcome data: down payment, monthly, total cost, timeline, effort, and risk. |
| 3 | Location must affect every path and every calculation. |
| 4 | picknbuild must include customization. |
| 5 | picknbuild must include title-type variation: clean vs rebuilt. |
| 6 | No dead ends. Every empty state must show an estimate, alternative, or next action. |
| 7 | All listings must pass through Reality Check before display. |
| 8 | All meaningful user actions must trigger a notification. |
| 9 | PicknBuild must have internal visibility into all high-intent actions. |
| 10 | All supply must be continuously ingested, refreshed, and normalized. |
| 11 | The platform must learn from outcomes, including likely off-platform sales. |
| 12 | Every action must produce a defined system response. |
| 13 | Every meaningful object in the system must be tied to a user profile, dealer page, seller page, or platform-owned listing. |
| 14 | The system must support both discovery and decision-making. |
| 15 | Nothing in the product may depend entirely on manual operations to function. |

## Expanded Notes (verbatim)

1. **Every vehicle must show all applicable acquisition paths.**
2. **Every path must show full outcome data:** down payment, monthly, total cost, timeline, effort, and risk.
3. **Location must affect every path and every calculation.**
4. **picknbuild must include customization.**
5. **picknbuild must include title-type variation:** clean vs rebuilt.
6. **No dead ends.** Every empty state must show an estimate, alternative, or next action.
7. **All listings must pass through Reality Check before display.**
8. **All meaningful user actions must trigger a notification.**
9. **PicknBuild must have internal visibility into all high-intent actions.**
10. **All supply must be continuously ingested, refreshed, and normalized.**
11. **The platform must learn from outcomes,** including likely off-platform sales.
12. **Every action must produce a defined system response.**
13. **Every meaningful object in the system must be tied to** a user profile, dealer page, seller page, or platform-owned listing.
14. **The system must support both discovery and decision-making.**
15. **Nothing in the product may depend entirely on manual operations to function.**
