# Routing System

The routing system defines what happens after every action. No user action should exist without a defined system outcome.

## Action-to-Outcome Mapping

| User Action | System Outcome |
| --- | --- |
| Save vehicle | Add to Garage + notify relevant parties |
| Start Now | Trigger payment + initiate process |
| Talk to Someone | Open call/message option + notify internal team |
| Dealer interest | Notify dealer + allow lead unlock |
| Seller message | Open internal chat |
| External deal submission | Parse + insert + run Reality Check |
| Feed engagement | Update user cluster and recommendations |

## Detailed Outcomes

- **Save vehicle** — add the vehicle to the user's Garage and notify relevant parties.
- **Start Now** — trigger payment and initiate the process.
- **Talk to Someone** — open a call or message option and notify the internal team.
- **Dealer interest** — notify the dealer and allow lead unlock.
- **Seller message** — open internal chat.
- **External deal submission** — parse the submission, insert it into the system, and run Reality Check.
- **Feed engagement** — update the user cluster and recommendations.

## Rule

> No user action should exist without a defined system outcome.
