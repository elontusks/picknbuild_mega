# Garage System

**Garage is not saved items. It is a decision engine.**

The Garage must be designed and operated as a tool that moves the user toward choosing a vehicle and path — not as a passive watchlist or bookmark folder.

---

## 13.1 Grouping

Vehicles in the Garage must be grouped by:

- Make
- Model
- Year

Grouping enables side-by-side comparison of the same vehicle family across different listings, paths, and sellers.

---

## 13.2 Comparison Table

For each grouped vehicle or vehicle family, the Garage must display a comparison table with the following columns:

| Column        | Description |
| ------------- | ----------- |
| Path          | Dealer, Auction, picknbuild, or Individual |
| Distance      | Distance from user to the vehicle / sourcing point |
| Down payment  | Path-specific down payment requirement |
| Monthly       | Realistic monthly payment |
| Total         | True total cost (all-in) |
| Timeline      | Expected time to ownership |
| Effort        | Effort level required from the user |
| Risk          | Risk level (low / medium / high) |

Every grouped row must populate all eight columns.

---

## 13.3 Decision Highlighting

The Garage must identify and **visually highlight** the following across the grouped rows:

- Best total cost
- Lowest monthly
- Fastest option
- Lowest risk
- Best fit for this user

These highlights must be derived from the Reality Check Engine's outputs, not from raw listing values.

---

## 13.4 Actions

From the Garage, the user must be able to:

- Start picknbuild
- Talk to someone
- Contact dealer
- Message seller
- Compare all paths again

Each action must be available directly from the Garage without requiring the user to navigate back to the original listing.

---

## Rule

**Garage must push the user toward a decision, not just hold items.** Every element of the Garage — grouping, comparison, highlighting, actions — exists to narrow the user's choice set and surface the best path forward.
