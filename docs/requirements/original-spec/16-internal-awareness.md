# 16. PicknBuild Internal Awareness System

PicknBuild must maintain visibility into every high-intent action that occurs on the platform so the team can intervene, guide, and close.

## Internal Notification Triggers

PicknBuild must receive internal notifications when:

- Users save dealer vehicles
- Users click into dealer listings
- Users show dealer interest
- Dealers receive leads
- Dealers unlock leads
- Users message sellers
- Sellers reply
- Conversations continue
- Users interact with auction listings
- Users paste external deals
- Users customize PicknBuild vehicles
- Users click Start Now
- Users click Talk to Someone

### Trigger summary

| Surface | Events that must notify PicknBuild |
| --- | --- |
| Dealer interactions | Save dealer vehicle; click into dealer listing; show dealer interest |
| Dealer lead flow | Dealer receives lead; dealer unlocks lead |
| Seller messaging | User messages seller; seller replies; conversation continues |
| Auction | User interacts with auction listings |
| External deal intake | User pastes external deal |
| Build path | User customizes PicknBuild vehicle |
| Primary CTAs | Click Start Now; click Talk to Someone |

## Internal Dashboard Views

PicknBuild must have an internal dashboard showing:

- Active interactions
- Deal progression
- High-intent users
- Conversation activity
- Lead flow

| View | Shows |
| --- | --- |
| Active interactions | Currently live user activity across surfaces |
| Deal progression | Stage of each user's decision journey |
| High-intent users | Users surfaced as most likely to transact |
| Conversation activity | Ongoing messaging across buyer/seller/dealer/team |
| Lead flow | Leads created, unlocked, and progressed |

## Governing Rule

> No high-intent action should happen without PicknBuild awareness.
