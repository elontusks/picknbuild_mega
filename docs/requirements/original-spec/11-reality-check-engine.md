# Reality Check Engine

**Core Decision System — Mandatory.**

---

## Purpose

The Reality Check Engine is the core intelligence layer of PicknBuild. Its purpose is to transform raw vehicle listings into personalized, decision-ready outcomes based on the user's real-world situation.

It does not evaluate the car alone. It evaluates:

- The user
- The path
- The cost
- The risk
- The outcome

---

## Core Principle

The Reality Check Engine answers one question:

> **"What does this actually mean for YOU?"**

Every output of the engine must be traceable back to that question.

---

## Input System

The engine must process all relevant user and vehicle data, including but not limited to the categories below.

### User Financial Inputs

- Income
- Savings
- Current cash available
- Monthly expenses (rent, bills, etc.)
- Employment status

### User Credit and Financing

- Credit score (or estimated range)
- Financing capability
- Down payment ability

### User Assets

- Current vehicle ownership
- Ability to liquidate assets
- Trade-in potential

### User Preferences

- Make, model, year
- Budget
- Timeline (fast vs flexible)
- Preference (lowest monthly vs lowest total vs customization)
- Willingness to repair / DIY capability

### User Context

- Location
- Driving needs
- Risk tolerance
- Whether they want to disclose full data or partial data

---

## Vehicle and Market Inputs

- Listing price
- Mileage
- Title type (clean vs rebuilt)
- Location
- Market value data (MarketCheck or equivalent)
- Auction behavior (if applicable)
- Estimated repair costs (if applicable)

---

## Output System

The engine must generate the following seven outputs.

### 1. True Total Cost

- All-in cost (not sticker price)
- Financing-adjusted cost based on user profile

### 2. Monthly Impact

- Realistic monthly payment
- Affordability relative to user income

### 3. Down Payment Requirements

- Path-specific down payment (dealer, picknbuild, etc.)

### 4. Risk Level

- Low / medium / high
- Based on title type, condition, financial strain, and user profile

### 5. Distance + Logistics

- How far the vehicle is
- Shipping or delivery implications

### 6. Path Comparison Intelligence

Which path is better for the user:

- Lower monthly
- Lower total
- Faster
- Less risk
- More customizable

### 7. Personalized Recommendation Signals

- "Better for you" indicators
- Warnings when something is financially risky

---

## Adaptive Behavior

The engine must dynamically adjust outputs based on:

- What the user chooses to disclose
- Changes in user inputs
- Behavior (pick, pass, save)
- Garage contents
- Historical interaction patterns

---

## Partial Data Handling

If the user does not provide full information, the system must:

- Estimate missing values
- Provide reasonable ranges
- Still generate usable outputs

### Rule

**The system must function even with incomplete data.** Missing inputs are never a valid reason to suppress outputs.

---

## System Influence

The Reality Check Engine must directly influence:

- Pricing displays
- Path rankings
- Recommendations
- Garage comparisons
- Feed relevance
- Notifications

Any surface that depends on these elements must read from the engine, not from raw listing data.

---

## Final Rule

**No vehicle, path, or recommendation may be shown without being processed through the Reality Check Engine.**

---

## Core Insight

The Reality Check Engine converts:

- Listings → decisions
- Prices → outcomes
- Options → clarity

It is the system that makes PicknBuild intelligent.
