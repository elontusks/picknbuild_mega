# 17. Feed and Social System

The feed is not generic social media. It is a decision-support community.

## 17.1 Purpose

- Surface real experiences
- Help discovery
- Build trust
- Generate supply
- Connect similar users

## 17.2 World of People Algorithm

The feed must cluster users to surface the most relevant content and peers.

### Clustering inputs

- Search behavior
- Pick/pass behavior
- Garage contents
- Financial reality
- Posting behavior
- Vehicle interests

### Potential clusters

- Budget-constrained buyers
- First-time buyers
- Auction DIY users
- PicknBuild users
- Deal hunters
- Customization/build users

### Prioritization

Feed should prioritize:

- Content from similar users
- Content relevant to similar cars
- Content relevant to the user's decision stage

## 17.3 Content Types

- Deal posts
- Problem/experience posts
- Question posts
- Build posts
- Recommendations
- Warnings

| Type | Purpose |
| --- | --- |
| Deal | Share a specific deal or listing |
| Problem/experience | Share lived experiences |
| Question | Request community input |
| Build | Document a PicknBuild-style build |
| Recommendation | Endorse a vehicle, path, or seller |
| Warning | Alert community to a risk or bad actor |

## 17.4 Feed → Decision Engine Integration

If a feed post contains a vehicle, the system must allow the user to:

- Click into the vehicle card
- Run Reality Check
- Show all acquisition paths

> Rule: Vehicle content in the feed must feed back into the decision engine.

## 17.5 Feed as Supply Engine

If users post vehicles or links they:

- Can become listings
- Can expand inventory
- Can improve matching

## 17.6 Micro-Dealer Participation

The feed must allow small dealers to:

- Post vehicles
- Build credibility
- Generate leads
- Participate without heavy onboarding

## 17.7 Feed Interaction

Users must be able to:

- Like
- Comment
- Reply
- Share

These interactions must trigger notifications.

## 29.11.1 Feed Media Support

The feed system must support full media functionality.

- Posts must support text, images, and video uploads.
- Deal posts must allow attached media.
- Build posts must allow before/after and progress media.
- Media must be viewable directly within the feed.

| Post type | Required media capability |
| --- | --- |
| Any post | Text, images, video |
| Deal post | Attached media |
| Build post | Before/after media; progress media |
| Rendering | Media must be viewable inline in the feed |

---

## 29.11.4 User Vehicle Upload Flow

Users must be able to create listings for vehicles they own. The system must support:

- Vehicle upload
- Price input
- Negotiable toggle
- Photo and video upload
- Option to make the listing public
- Ability to receive messages

| Step | Requirement |
| --- | --- |
| 1 | Vehicle upload |
| 2 | Price input |
| 3 | Negotiable toggle |
| 4 | Photo and video upload |
| 5 | Option to make listing public |
| 6 | Ability to receive messages |
